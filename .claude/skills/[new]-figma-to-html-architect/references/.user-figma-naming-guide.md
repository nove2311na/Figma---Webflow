# Figma Layer Naming Guide for HTML Generation

To ensure the AI system correctly generates Semantic HTML tags (like `<button>`, `<form>`, `<input>`) instead of generic `<div>` tags, you **MUST** include specific keywords in your Figma Layer names.

When exporting, Figma creates a `data-name` attribute based on your layer name. The system will look for the following exact keywords anywhere in the layer name (e.g., `Header / nav_menu` or `form_wrapper_contact`).

## Required Keywords & Resulting Tags

| Figma Keyword | Resulting HTML Tag | Description |
|---|---|---|
| `nav_menu` | `<nav>` | Use for navigation blocks |
| `button` | `<button>` | Use for clickable buttons |
| `form_wrapper` | `<form>` | Use for the outer form block |
| `form_input` | `<input>` | Use for text input fields |
| `form_textarea`| `<textarea>` | Use for large text areas |
| `form_submit` | `<button type="submit">`| Use for the form submit button |
| `image` | `<img>` | Use for images |
| `paragraph` | `<p>` | Use for text blocks |
| `heading_h1` | `<h1>` | Use for H1 headings |
| `heading_h2` | `<h2>` | Use for H2 headings |
| `list_wrapper` | `<ul>` / `<ol>` | Use for lists |
| `list_item` | `<li>` | Use for items inside lists |
| `section` | `<section>` | Use for major page sections |
| `header` | `<header>` | Use for page headers |
| `footer` | `<footer>` | Use for page footers |

If you just use a generic name like `Frame 1234` or `Group 1`, the system will either assign a generic `<div>` or throw a validation error.
