# Gotcha 04 — Native Class Denylist

**Webflow ships native classes** that power its built-in widgets. Prefixes: `.w-*` and `.wf-*`.

Examples from the user's CSS:
- `.w-checkbox` (form checkbox widget)
- `.w-checkbox-input` (the actual input element)
- `.w-form-formradioinput--inputType-custom`

These classes have **Webflow-defined styles** that override anything you write. You can't redefine `.w-checkbox` — it will be regenerated by the widget.

## Trap

1. A Figma layer is named `w-checkbox` (or contains "checkbox"). The processor maps it to a class name `w-checkbox`. Webflow sees this as a native widget class and applies its own styles, ignoring your definitions.

2. A Figma class is named `button` (no prefix). The processor creates a `button` class. Webflow Designer might apply built-in `.button` styles from a template — collision.

3. `wf-form` is sometimes used by Figma templates for "Webflow Form" — collision with the real `.wf-form`.

## Detection

The `webflow-native-class-index.json` (under `knowledge-base/generated/`) lists all known native classes. The `selector_guards.is_native_selector()` function in `.claude/skills/_shared/` checks against this list.

```python
from _shared.selector_guards import is_native_selector

is_native_selector("w-checkbox")           # → True
is_native_selector("button")                # → False (no prefix, but check anyway)
is_native_selector("wf-form")               # → True
is_native_selector("card_component")        # → False
```

## Code-level rule

Any class name matching a denylist entry must either:
- Be renamed (e.g. `button` → `button_component`).
- Or the build halts and the user is asked to resolve.

The `process_html.py` script (in `figma-to-html-architect`) calls `is_native_selector()` on every output class. Matches are flagged for user review.

## Fix

1. Maintain `webflow-native-class-index.json` as the source of truth for denylist.
2. Update it whenever Webflow ships a new widget.
3. In the LLM prompt for class naming, add the denylist as a hard constraint.
4. Validation gate (`validate_figma_extraction.py`) rejects any contract mapping to a denylist entry.
